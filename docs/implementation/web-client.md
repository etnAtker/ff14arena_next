## 1. 页面结构

当前客户端包含以下页面：

- 首页
- 模拟页
- 服务器性能观测页

当前切换规则如下：

- 未进入房间时显示首页
- 进入任意房间后直接显示模拟页
- 顶栏“观测”按钮会跳转到独立路径 `/metrics`
- `/metrics` 只渲染服务器性能观测页，不加载大厅数据，也不创建模拟页输入循环

首页房间列表中，每个房间提供“加入”和“加入观战”入口。
“加入”按普通玩家身份请求可用槽位；“加入观战”直接进入观战席，不占用战斗槽位，槽位满员时仍可进入等待态房间。

## 2. 页面分工

当前前端页面壳层分工如下：

- `App.vue`
  负责 provider、顶层状态、键盘输入、镜头状态与页面切换
- `components/layout`
  负责顶栏等全局壳层组件
- `components/pages`
  负责首页、模拟页和服务器性能观测页页面组件
- `components/battle`
  负责战斗场地、镜头换算与场景绘制
- `utils/ui.ts`
  负责 phase 标签、槽位颜色、角色简写等展示辅助逻辑

当前页面与资源加载策略如下：

- 页面组件与战斗视图按需异步加载
- 首屏完成后，客户端会后台预加载其余页面模块、战斗渲染模块和 Socket 客户端依赖
- Socket 连接仍在首次需要联机时才创建
- 进入房间或战斗开始时，客户端会读取当前战斗静态数据，并预加载该战斗状态图标；图标加载失败会在页面顶部显示错误

## 3. 操控模式

当前客户端支持三种本地操控模式：

- 传统
- 标准
- 固定

该设置保存在浏览器本地存储中。

### 传统模式

- 左键拖拽改变镜头朝向
- 右键拖拽改变镜头朝向
- 左键和右键拖拽都不会直接改变人物面向
- `WASD` 移动方向以当前镜头朝向为准
- 人物在移动时自动朝向当前移动方向

### 标准模式

- 左键拖拽只改变镜头朝向
- 右键拖拽同时改变镜头朝向和人物朝向
- 右键拖拽时，人物面向与当前镜头方向保持一致
- `WASD` 移动方向以人物当前面向为准
- 移动本身不会修改人物面向

### 固定模式

- 鼠标拖拽不会改变镜头朝向
- 地图固定绘制在画面正中心
- 地图不随玩家位置移动，也不随镜头旋转
- `WASD` 移动方向以固定地图坐标为准
- 人物在移动时自动朝向当前移动方向

### 通用输入

- 键盘 `1` 触发防击退
- 键盘 `2` 触发冲刺
- 连续覆盖型输入按固定间隔发送
- 房间操作与防击退、冲刺这类一次性命令仍保持独立事件

## 4. 输入与同步

当前客户端同步状态集中在 `stores/app.ts`。

### 当前输入链

- 连续覆盖型输入统一通过 `sim:input-frame` 发送
- 输入帧包含：
  - `position`
  - `facing`
  - `moveDirection`
  - `syncId`
  - `issuedAt`
- 客户端所有上行模拟输入都携带当前 `syncId`
- 客户端先本地推进自己的移动，再把当前位姿样本上报给服务端
- 冲刺状态存在时，客户端本地移动按 `1.3` 倍移动速度推进
- 客户端收到本机角色的强制位移请求后，在本地执行位移并通过 `sim:input-frame` 回传结果
- 客户端触发防击退或冲刺时，会先按共享冷却常量写入本地乐观 CD
- 每次收到 `sim:snapshot` 时，客户端直接以服务端快照中的权威 CD 覆盖本地乐观 CD
- `waiting` 与 `running` 永远共用同一套 `sim:input-frame` 发送链路

### 当前快照链

- `sim:start` 下发一轮新同步流的起始快照
- `sim:events` 下发当前同步流中的增量事件
- `sim:snapshot` 下发 join / rejoin / resync 快照以及周期性权威快照
- 每一轮快照与事件都带 `syncId`
- 客户端只接受当前 `syncId` 的同步流，旧 `syncId` 数据会被丢弃
- 客户端发现房间状态已进入 `running` 但本地还没有运行态快照时，会主动请求重同步

### 当前重同步行为

- 客户端在缺失权威快照或怀疑相位错位时，会主动发送 `sim:request-resync`
- 服务端会向该连接回送当前权威快照、房间状态和槽位状态
- 运行中断线后，客户端重新加入房间会收到当前权威快照

### 当前本地预览边界

- 当前已实现本地朝向预览
- 当前已实现本机连续移动的即时乐观移动
- 客户端会在收到快照后，把本机最新本地位姿重新覆盖到当前镜像上
- 客户端会忽略本机角色的 `actorMoved` 回包，本机位置由本地输入与本地强制位移执行结果决定
- 上述本地移动与位姿覆盖逻辑在 `waiting` 和 `running` 两种 phase 中共用同一套实现
- 客户端不再依赖输入确认序号、时间同步或未确认输入重放
- 当前服务端不对玩家上传位姿样本执行移动速度上限校验
- 远端角色使用渲染层位置镜像平滑追逐权威位置
- 远端角色距离权威位置过大时，渲染层会直接跳到权威位置，不做平滑
- 当前尚未实现更细致的特殊位移预测

## 5. 镜头与绘制

当前战斗场地绘制使用 PixiJS。

当前镜头行为如下：

- 传统模式和标准模式下，镜头朝向由鼠标拖拽控制；固定模式下，鼠标拖拽不改变镜头朝向
- 进入房间、切换受控角色或切换战斗时，初始镜头朝向与当前受控角色面向一致
- 倒计时结束正式开始模拟时，镜头朝向和缩放保持不变
- 模拟结束回到等待态时，镜头朝向保持不变
- 鼠标滚轮控制缩放
- 滚轮缩放事件在渲染层通过 `requestAnimationFrame` 合并后再同步给 Vue
- 双击场地重置缩放到默认值
- 传统模式和标准模式下，当前受控玩家始终绘制在画面中心点偏下固定位置
- 固定模式下，场地中心始终绘制在画面正中心
- 当没有可控角色时，画面回到场地中心绘制

当前场地会绘制：

- 场地边界
- Boss 目标环
- 地图标点
- Boss 本体
- 玩家与 Bot 位置点及职能简写
- 朝向指示线
- 圆形 / 环形 / 分摊 / 分散 AOE
- 圆形攻击预兆
- 塔判定范围
- 首领到玩家的连线
- 顶部 Boss 读条覆盖层，读条进度使用 CSS 线性动画保持连续
- 同一条 Boss 读条只在首次进入前端视图时建立动画锚点，后续同读条快照不会重置 CSS 动画时间轴

## 6. HUD 与信息展示

当前模拟页左栏显示：

- 8 个固定槽位卡片，按双列展示
- 小队卡片显示顺序可在本地调整，调整只影响客户端展示，不改变服务端槽位顺序和机制判定顺序
- 槽位名、成员名与切换 / 开始按钮
- 切换槽位或从观战入场前需要二次确认
- 观战状态下，点击可用槽位回到场内
- 当前 HP / 最大 HP
- 按职能区分的底色与自己的强调色
- 机制状态图标，按 XIVAPI `PartyListPriority` 排序，图标下方显示状态倒计时
- 状态图标固定按 `24px x 32px` 展示；缺少图标或图标加载失败时显示最多两个字的兜底文字
- 主战斗区域左下角显示自身状态 HUD，展示形式与小队列表状态图标一致
- 防击退与冲刺按钮显示当前 CD 状态
- 技能就绪时显示 `防击退（1）`、`冲刺（2）`
- 技能冷却中显示剩余秒数，例如 `防击退 12.3s`
- 技能按钮禁用状态同时检查战斗阶段、受控角色、角色存活和对应 CD
- 房主在运行态可点击“快速失败”，客户端发送房间操作请求，由服务端立即结算本轮失败

当前模拟页右栏显示：

- 上一轮结果摘要
- 失败原因列表
- 当前 8 人状态摘要
- 实时战斗日志

当前等待态与战斗中都使用 `SimulationSnapshot` 作为主要展示数据：

- `waiting` 阶段用于进房预览、切槽和等待开始
- `running` 阶段用于权威模拟展示
- `latestResult` 用于右侧栏展示上一轮结果

场地标题栏在操控模式选择左侧显示观战按钮。
等待态中，占槽玩家点击后切换为观战，原槽位由 Bot 接管。
普通玩家进入观战后观战按钮置灰；房主进入观战后该按钮显示为“开始”，并可直接发起开始倒计时。
房主可在场地标题栏配置开始倒计时，默认 `5s`，范围为 `1s` 到 `30s`。
所有客户端根据服务端 `room:countdown` 显示整秒倒计时；正式收到 `sim:start` 后显示 `战斗开始！` 持续 `1s`。
倒计时开始时场景会回到标准开场状态；倒计时期间玩家的移动和朝向会保留到正式开战。
观战成员不控制任何角色，不发送移动、防击退或面向输入，但仍接收房间状态、快照和战斗事件。
客户端点击观战后会立即进入本地观战等待态，先清除本地受控角色位姿与面向预览，并在服务端房间状态回包前停止发送移动、技能和强制位移回传输入，避免旧槽位控制权已经释放后继续向服务端发送旧角色输入。

HUD 文本时钟以 `100ms` 间隔刷新。状态倒计时和技能 CD 文案使用该低频时钟；开始倒计时数字由服务端整秒广播；Boss 读条进度不依赖 Vue 高频刷新，而是由 CSS 动画连续推进。同一读条期间，Vue 不会因为事件或快照时间变化反复改写读条动画进度。

## 7. 性能观测页

当前性能观测页通过 `GET /admin/metrics` 轮询服务端短期内存指标。

页面行为如下：

- 默认每 `2` 秒拉取一次指标
- 只读取聚合后的指标 JSON
- 不建立额外 Socket 订阅
- 不触发服务端落盘
- 与首页和模拟页状态解耦

页面展示以下信息：

- Socket 当前连接数、累计连接数和断开数
- 房间总量、运行中房间数、活跃模拟数、在线玩家数和 Bot 数
- Tick p95、Tick 最大耗时、Tick 超时次数
- Bot controller p95、Bot 控制帧数量
- core simulation tick p95
- HTTP 路由请求数、错误率、请求速率、耗时 p95 和最大耗时
- Socket 上行事件、下行事件和错误码分布
- 房间级玩家数、Bot 数、Tick、输入帧、丢弃旧输入、事件、快照和重同步统计
