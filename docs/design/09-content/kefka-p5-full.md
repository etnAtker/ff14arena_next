# 凯夫卡P5：整合

本文档描述凯夫卡 P5 整合练习内容。  
本战斗规则实现于 `packages/content`，前端与服务端不得实现第二套战斗规则。

## 战斗范围

- 战斗 id：`kefka_p5_full`
- 战斗名称：`凯夫卡P5：整合`
- 首领：`凯夫卡`
- 小队：8 人，标准 `MT/ST/H1/H2/D1/D2/D3/D4`
- 场地半径 `20m`，首领目标圈半径 `8m`
- 场地背景图使用 `/map/kefka-p5.png`，图片为 `2000px x 2000px`，按 `50px = 1m` 映射为 `40m x 40m`，中心对齐场地中心
- 标点和初始站位沿用凯夫卡 P5 现有配置

## 时间轴来源

整合战以 `tmp/P5施法.csv` 和 `tmp/P5buff.csv` 为时间来源。  
带读条时间的日志行按判定时点处理，读条开始时间由判定时点减去读条时间得到。

`T+0` 定义为第一条连续究极读条开始，对应日志中 `15:05.799 连续究极 3.70 秒` 的 `3.70s` 前。

## 跳时预设

凯夫卡 P5 整合只开放以下开始时间：

- `0ms`：从头
- `16550ms`：洪水
- `28750ms`：癫狂交响曲
- `50450ms`：三星
- `95350ms`：地火
- `146250ms`：遗弃末世

“从头”从 `0ms` 开始。洪水跳时点为洪水读条开始所在 tick；遗弃末世跳时点为遗弃末世读条开始 tick；其它跳时点为对应机制读条开始前 `1s`，并按固定 tick 对齐到 `50ms`。  
跳时启动时不重设玩家位置，正式开战时保留倒计时结束位置。已跳过机制的 Buff、场地物和读条不做额外重建。

## 当前实现取舍

- 连续究极为无效果普通 AOE。
- `unknown_c652` 忽略，不生成机制。
- `unknown_c183` 作为洪水预兆来源处理。
- 魔击按癫狂交响曲后续职能分摊站位处理。
- 二选一灾祟保持随机，不固定为日志中的两次龙卷结果。
- 混沌末世只有开头一次长读条；地火批次按日志修正为 `100132ms / 102674ms / 105173ms / 107669ms / 110161ms / 112658ms` 开始。
- 每批地火首判定保持日志修正时点后 `625ms`；初始预兆从首判定前 `4000ms` 出现，预兆内显示斜向步进方向箭头；之后每 `500ms` 判定一次，共 `7` 次，判定后显示 `300ms` 范围。
- 混沌末世地火命中仍按训练规则即死，暂不改为日志中的伤害降低。
- 混沌涡旋实现为 8 人 `5m` 分散。
- 最后一组魔击后追加遗弃末世；四轮遗弃末世每轮同时出现紫圈、黄圈和 8 人分摊。紫圈全程不重复，判定后转为持久场地残留；黄圈按预兆出现时玩家最近候选点随机选择，判定后不残留；分摊少于 8 人参与时失败。
- Bot 已登记，使用分段状态机处理整合流程；地火和三星随机路线仍按固定点策略处理；地火结束后才切换到混沌涡旋的半径 `8m` 就近分散点；遗弃末世使用半径 `10m` 圆上的 4 个固定斜向等待点和固定圆弧路径。第三轮默认顺时针，只有第三轮正点紫圈挡住本轮顺时针路径时改逆时针；第四轮沿第三轮路径反向回到第三轮起跑等待点。
