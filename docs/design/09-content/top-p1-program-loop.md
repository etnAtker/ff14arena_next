# 欧米茄绝境战 P1：循环程序

本文档描述《最终幻想 XIV》绝本“欧米茄绝境战”P1 首个主要机制“循环程序”的模拟设计。英文资料名为 `The Omega Protocol (Ultimate)` / `Program Loop`。

当前目标是做机制练习内容，不做完整副本复刻。  
本文档中的机制规则应实现于 `packages/core` 与 `packages/content`，前端与服务端不得实现第二套战斗规则。
所有对玩家展示的技能、状态、提示与失败原因文案必须使用中文。英文名称仅作为资料原名、内部标识或代码注释使用，不进入玩家可见界面。

## 资料来源

- Icy Veins: The Omega Protocol Ultimate Phase 1 Guide
- Thaliak: The Omega Protocol (Ultimate)
- FFXIV Wiki: The Omega Protocol (Ultimate)
- Tuufless's FFXIV Notes: Omega / Program Loop

上述资料对核心机制描述一致：塔按 `1 -> 2 -> 3 -> 4` 处理，连线按 `3 -> 4 -> 1 -> 2` 处理；塔和连线都会附加破灭刻印，重复吃到会转为死亡宣告；连线额外附加体力衰减。

## 核心机制流程

### 总览

1. 首领读条循环程序。
2. 读条结束时，8 名玩家获得循环程序状态与第一至第四目标。
3. 每个编号随机分配给 2 名玩家。
4. 场上出现 2 座塔，同时首领给 2 名玩家生成连线。
5. 之后共进行 4 轮结算，每轮同时结算：
   - 当前轮的 2 座塔
   - 当前持有的 2 条连线
6. 塔处理顺序为第一目标、第二目标、第三目标、第四目标。
7. 连线处理顺序为第三目标、第四目标、第一目标、第二目标。
8. 第 4 轮结算后，若没有失败结果，机制完成。

### 时间轴

时间以战斗开始后毫秒计。

| 时间      | 事件                        | 数据                             |
| --------- | --------------------------- | -------------------------------- |
| `0ms`     | 战斗开始                    | 首领位于中心                     |
| `6000ms`  | 首领开始读条循环程序        | 读条 `4000ms`                    |
| `10000ms` | 循环程序读条完成            | 分配循环程序状态与第一至第四目标 |
| `10000ms` | 第 1 轮塔生成，初始连线生成 | 塔编号 1，连线目标随机           |
| `10000ms` | 首领开始第 1 次冲击波读条   | 读条 `7600ms`，结束于第 1 轮结算 |
| `17600ms` | 第 1 轮结算                 | 1 号踩塔，3 号吃线               |
| `17600ms` | 第 2 轮塔生成               | 塔编号 2                         |
| `19000ms` | 首领开始第 2 次冲击波读条   | 读条 `7600ms`，结束于第 2 轮结算 |
| `26600ms` | 第 2 轮结算                 | 2 号踩塔，4 号吃线               |
| `26600ms` | 第 3 轮塔生成               | 塔编号 3                         |
| `28000ms` | 首领开始第 3 次冲击波读条   | 读条 `7600ms`，结束于第 3 轮结算 |
| `35600ms` | 第 3 轮结算                 | 3 号踩塔，1 号吃线               |
| `35600ms` | 第 4 轮塔生成               | 塔编号 4                         |
| `37000ms` | 首领开始第 4 次冲击波读条   | 读条 `7600ms`，结束于第 4 轮结算 |
| `44600ms` | 第 4 轮结算                 | 4 号踩塔，2 号吃线               |
| `47500ms` | 机制结束判定                | 成功或失败                       |

说明：

- 资料页列出 Blaster 读条 `7.6s`，并将每轮 Blaster 与 Storage Violation 作为同轮次结算事件。
- 为便于练习脚本从开场直接进入机制，本文档将读条开始设为 `6000ms`，让第一轮结算落在 `17600ms`。
- 冲击波使用 `7600ms` 读条，每次读条结束与该轮塔和冲击波结算同 tick。
- 后续轮次按 `9000ms` 间隔推进，贴合循环程序状态持续时间差值 `9s`。

## 状态与技能数据

### 循环程序状态

| 编号     | 玩家数 | 持续时间 | 塔处理轮次 |
| -------- | -----: | -------: | ---------: |
| 第一目标 |      2 |    `16s` |    第 1 轮 |
| 第二目标 |      2 |    `25s` |    第 2 轮 |
| 第三目标 |      2 |    `34s` |    第 3 轮 |
| 第四目标 |      2 |    `43s` |    第 4 轮 |

规则：

- 每个玩家获得一个循环程序编号。
- 持有循环程序状态的玩家可以踩塔。
- 塔只接受仍持有循环程序状态的玩家。
- 成功踩塔后移除该玩家的循环程序状态与目标编号。
- 循环程序状态自然到期且未被移除时，玩家获得遗忘。

### 遗忘

- 来源：循环程序状态到期未解除
- 持续时间：不单独模拟
- 效果：直接死亡
- 练习处理：获得该状态时立即写入失败原因并击杀玩家

说明：资料页写明遗忘会导致玩家无法执行动作并随机移动，实战上通常等同机制失败。第一版模拟不实现随机失控移动，直接按死亡处理。

### 冲击波

| 项       |                             数值 |
| -------- | -------------------------------: |
| 读条时间 |                         `7600ms` |
| 结算时机 |                       读条结束时 |
| 目标数   |                                2 |
| 伤害     |                          `35847` |
| 范围形状 | 以当前连线持有者为中心的圆形范围 |
| 范围半径 |                            `15m` |
| 预兆时间 |                          `500ms` |
| 附加状态 |               体力衰减、破灭刻印 |

规则：

- 两条连线在第 1 轮开始时生成。
- 连线不会在结算后消失，会保留到第 4 轮结束。
- 连线可以被其他玩家穿过并接走。
- 每轮冲击波结算前 `500ms`，以当前连线持有者位置显示圆形攻击预兆。
- 每轮冲击波结算时，以当前连线持有者为中心生成圆形伤害。
- 连线持有者与范围内其他玩家均受到伤害与状态。

说明：冲击波范围按首领目标环处理，半径为 `15m`。

### 体力衰减

| 项             |                     数值 |
| -------------- | -----------------------: |
| 持续时间       |                 `9000ms` |
| 最大生命值修正 | 第一版不模拟具体血量修正 |
| 生效时机       |         冲击波伤害结算后 |

规则：

- 先结算冲击波伤害，再附加体力衰减。
- 第一版不模拟最大生命值降低、当前生命值夹取或治疗补回。
- 持有体力衰减的玩家受到任何非 0 机制伤害时直接死亡。
- 体力衰减附加时不额外模拟血量下降。
- 3 号与 4 号吃线后，在其踩塔前体力衰减会结束；第一版只要求其不要在体力衰减期间再受伤。

### 破灭刻印

| 项           |                      数值 |
| ------------ | ------------------------: |
| 持续时间     |                 `10000ms` |
| 最大安全层数 |                         1 |
| 失败阈值     | 获得第 2 层时转为死亡宣告 |

规则：

- 冲击波与塔都会附加 1 层破灭刻印。
- 若玩家在状态未消失前再次获得破灭刻印，立即获得死亡宣告。
- 机制通过塔顺序与连线顺序错开，避免同一玩家连续承受两种判定。

### 死亡宣告

- 来源：破灭刻印达到 2 层
- 持续时间：不单独模拟
- 效果：直接死亡
- 练习处理：获得该状态时立即写入失败原因并击杀玩家

### 塔判定

| 项       |                  数值 |
| -------- | --------------------: |
| 来源     |                塔结算 |
| 每轮塔数 |                     2 |
| 伤害     |               `35847` |
| 附加状态 |              破灭刻印 |
| 成功人数 | 每塔至少 1 名有效玩家 |

规则：

- 每座塔必须由 1 名仍持有循环程序状态的玩家处理。
- 没有循环程序状态的玩家站在塔内不计入有效处理者。
- 有效处理者不足 1 人时，塔爆炸并团灭。
- 有效处理者为 1 人或多人时，所有有效处理者都受到塔伤害、获得破灭刻印，并移除循环程序状态与目标编号。

说明：资料描述塔必须由带循环程序状态的玩家处理，且不持有循环程序状态的玩家不计入塔处理。

## 塔位

循环程序共有 8 个可能塔位。  
塔位分布在四个基本方向两侧，每个基本方向 2 个点。

塔位按“正方向锚点 + 切线偏移”生成：

- 正方向锚点半径：`15m`
- 每个正方向生成 2 座塔，位于该正方向两侧
- 两座塔的中心连线为场地圆的切线
- 每座塔中心相对正方向锚点沿切线偏移 `3m`
- 同一正方向的两座塔中心间距为 `6m`

坐标如下：

| 塔位 id | 正方向 | 切线偏移 | 塔中心坐标  |
| ------- | ------ | -------: | ----------- |
| `N_W`   | 北     |    `-3m` | `(-3, -15)` |
| `N_E`   | 北     |     `3m` | `(3, -15)`  |
| `E_N`   | 东     |    `-3m` | `(15, -3)`  |
| `E_S`   | 东     |     `3m` | `(15, 3)`   |
| `S_E`   | 南     |    `-3m` | `(3, 15)`   |
| `S_W`   | 南     |     `3m` | `(-3, 15)`  |
| `W_S`   | 西     |    `-3m` | `(-15, 3)`  |
| `W_N`   | 西     |     `3m` | `(-15, -3)` |

当前实现每局随机生成塔序列：

- 从 8 个可能塔位中随机配对。
- 每轮取 2 个塔位，两塔在环形塔位索引上的差值必须为 `2`、`4` 或 `6`。
- 一局内 8 个塔位各出现 1 次，不重复。
- 塔处理编号仍按第一目标、第二目标、第三目标、第四目标推进。
- 同一编号内按 `H1 -> MT -> ST -> D1 -> D2 -> D3 -> D4 -> H2` 判定高低优先级。
- 每轮双塔从 A 点左侧开始顺时针排序：`N_W -> N_E -> E_N -> E_S -> S_E -> S_W -> W_S -> W_N`。
- 排序后的第一座塔由高优先级玩家处理，第二座塔由低优先级玩家处理。

参数：

- 正方向锚点半径：`15m`
- 切线偏移距离：`3m`
- 塔判定半径：`3m`
- 塔可视半径：`3m`

## 连线传递

### 初始连线

- 第 1 轮开始时，首领生成 2 条连线。
- 当前实现会随机分配循环程序编号。
- 初始目标为本局第二目标的两名玩家，高优先级玩家持有第 1 条线，低优先级玩家持有第 2 条线。
- 初始目标不是最终吃线责任者；玩家需要通过移动把线交给对应编号。

### 传递规则

- 连线以首领与当前持有者之间的线段表示。
- 其他存活玩家当前坐标或本 tick 移动路径与该线段相交时，可接走连线。
- 同一条连线传递后进入 `500ms` 冷却，避免同帧抖动。
- 每名玩家同一时刻最多持有 1 条连线。
- 若多个玩家同帧满足接线条件，按小队顺序选择。
- 当前 TOP 跑法按本局随机结果和优先级把两条线拆为两个 Bot 车道序列：`第二目标高优先级 -> 第三目标高优先级 -> 第四目标高优先级 -> 第一目标高优先级 -> 第二目标高优先级`、`第二目标低优先级 -> 第三目标低优先级 -> 第四目标低优先级 -> 第一目标低优先级 -> 第二目标低优先级`。Bot 按序列只允许下一位接线，并通过 Bot 专用冷却避免同轮连续传递；真实玩家不受 Bot 车道限制，实际穿过连线即可接到。

### 连线结算顺序

| 轮次 | 塔处理编号 | 连线处理编号 |
| ---: | ---------- | ------------ |
|    1 | 第一目标   | 第三目标     |
|    2 | 第二目标   | 第四目标     |
|    3 | 第三目标   | 第一目标     |
|    4 | 第四目标   | 第二目标     |

失败条件：

- 当前轮结算时，任意连线持有者不是本轮指定连线编号。
- 两条连线由同一个玩家持有。
- 冲击波命中正在踩塔的玩家。
- 冲击波互相重叠时不单独写特殊死亡规则，死亡判定统一按破灭刻印叠层处理：破灭刻印达到 2 层或以上时转为死亡宣告。

## 跑法约定

- 当前轮踩塔的 2 人去塔。
- 当前轮吃线的 2 人把线带到远离塔的位置。
- 其他人靠近塔位附近但避开冲击波范围。
- 吃线编号比踩塔编号晚两步：`3 -> 4 -> 1 -> 2`。
- 无职责玩家在场中安全点待机，避开当前塔位与冲击波范围。
- 连线玩家固定在半径 `17m` 的正点吃判定，正点按 `A -> B -> C -> D` 顺序分给高低优先级，且避开当前塔组。
- Bot 接到当前轮连线后，先停在最终冲击波点同方向、距离中心约 `10m` 的中转点；距离判定约 `2000ms` 时再前往半径 `17m` 的最终点。
- 踩塔玩家站在塔中心。

Bot 当前是确定性脚本，不实现完整智能决策。Bot 从自身循环程序状态和 `scriptState` 中的 Bot 车道序列判断职责：

- 当前编号等于塔编号：读取本局随机塔序列，移动到分配塔中心。
- 当前编号等于连线编号：穿过对应车道的当前连线，接线后先到 `10m` 中转点等待，临近判定再移动到外侧吃冲击波。
- 否则：移动到安全待机点。
- 3/4 号吃线后，在体力衰减结束前避免进入任何伤害范围。

## 失败文案

机制失败时调用 `state.fail(reason)`，不中断后续脚本，最终统一结算。

- 循环程序状态到期未解除：`{actor} 未及时踩塔解除循环程序`
- 错号踩塔：`{actor} 在错误轮次踩塔`
- 塔无人处理：`第 {round} 轮塔未被处理`
- 多人踩同塔：`第 {round} 轮塔被多人处理`
- 连线错误编号：`第 {round} 轮连线由错误编号处理`
- 冲击波命中塔组：`第 {round} 轮冲击波命中踩塔玩家`
- 破灭刻印叠层：`{actor} 重复承受循环程序判定并获得死亡宣告`
- 体力衰减期间受伤：`{actor} 在体力衰减期间受到伤害`
- 越界死亡：`{actor} 触碰死亡墙`
