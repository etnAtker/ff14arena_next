# 凯夫卡P2：一运

本文档描述《最终幻想 XIV》凯夫卡 P2 一运练习内容的模拟设计。

本战斗规则实现于 `packages/content`，前端与服务端不得实现第二套战斗规则。  
当前版本实现机制判定与调试用 Bot 跑法。  
调试 Bot 是固定路线，不实现完整智能重规划。

## 战斗范围

- 战斗 id：`kefka_p2_first_forsaken`
- 战斗名称：`凯夫卡P2：一运`
- 首领：`凯夫卡`
- 小队：8 人，标准 `MT/ST/H1/H2/D1/D2/D3/D4`
- 场地半径：`20m`
- 首领目标圈半径：`6m`

## 时间轴

时间以战斗开始后毫秒计。

| 时间       | 事件                    | 数据                         |
| ---------- | ----------------------- | ---------------------------- |
| `3000ms`   | 开始读条遗弃末世        | 读条 `6700ms`                |
| `9700ms`   | 遗弃末世读条完成        | 首次赋予头标，显示 `5000ms`  |
| `12700ms`  | 第 1 轮双塔生成         | 塔窗口 `10000ms`             |
| `22700ms`  | 第 1 轮双塔判定         | 判定头标，赋予第 2 轮头标    |
| `23700ms`  | 开始读条未来/过去终结   | 读条 `6100ms`                |
| `22700ms`  | 第 2 轮双塔生成         | 塔窗口 `10000ms`             |
| `32700ms`  | 第 2 轮双塔判定         | 近场 4 人点名，外 3 人出小怪 |
| `38000ms`  | Boss 与小怪读条消灭之脚 | 读条 `4700ms`                |
| `42700ms`  | 第 3 轮双塔判定         | 同时结算第 2 轮消灭之脚      |
| `62700ms`  | 第 4 轮消灭之脚结算     | 继续后续双塔                 |
| `82700ms`  | 第 6 轮消灭之脚结算     | 继续后续双塔                 |
| `102700ms` | 第 8 轮消灭之脚结算     | 机制完成                     |

第 2 到第 8 轮塔在上一轮塔判定时立即生成，并在 `10000ms` 后判定。

## 点名池

点名池初始数量：

- 分摊：`8`
- 大圈：`12`
- 扇形：`12`

首次赋予：

- 分摊：`1` 名 T/H 与 `1` 名 DPS
- 大圈：剩余 `3` 名 T/H 或剩余 `3` 名 DPS
- 扇形：另一组剩余 `3` 人

后续赋予：

- 奇数轮塔后：`2` 大圈 + `2` 扇形
- 偶数轮塔后：`2` 分摊 + `1` 大圈 + `1` 扇形
- 若指定点名数量不足，按 `分摊 -> 大圈 -> 扇形` 的 fallback 顺序补足
- 第 7 轮塔判定后不再赋予头标

头标视觉显示 `5000ms`，逻辑点名保留到该玩家踩塔判定。  
未踩塔玩家保留旧点名，不赋予新点名。

## 塔

塔可能位置共 `8` 个，中心距场中 `8m`，从正北开始每 `45°` 一个。  
每轮生成两座塔，塔半径 `4m`。
共 `8` 轮塔。

第 1 轮随机生成两座中心夹角 `90°` 的塔。  
之后随机一次顺时针或逆时针方向，每轮两座塔整体旋转 `45°`。

每座塔必须由 `2` 人处理。  
多于 `2` 人时随机选 `2` 人判定。  
少于 `2` 人时记录失败；若塔内有 `1` 人，该玩家死亡。

## 头标判定

所有机制伤害命中时，若目标已有 `injury_up`，目标即死。  
若目标没有 `injury_up`，受到伤害并获得 `3000ms` 易伤。

头标伤害在判定前 `500ms` 锁定范围并显示 `500ms` 预兆。

分摊：

- 半径 `5m`
- 包含点名本人
- 需要正好 `3` 人参与
- 范围内所有人受到伤害并获得易伤

大圈：

- 半径 `5m`
- 范围内所有人受到伤害并获得易伤

扇形：

- 点名者不参与命中判定
- 朝除点名者外最近的人释放
- 角度 `90°`
- 半径 `20m`
- 扇形内玩家受到伤害并获得易伤

## 未来/过去与消灭之脚

奇数轮塔判定后 `1000ms`，Boss 随机读条未来终结或过去终结。  
该结果用于紧随其后的偶数轮机制。

偶数轮塔判定前 `500ms`，选中距离场中最近的 `4` 名存活玩家，锁定以这些玩家为中心的半径 `5m` AOE，并显示 `500ms` 圆形预兆。  
偶数轮塔判定同时，锁定 AOE 范围内玩家受到伤害并附加易伤。  
这 `4` 人中距离场中较远的 `3` 人用于确定小怪生成方向，小怪没有目标圈。  
若目标距离场中小于等于小怪虚拟目标环半径 `4m`，小怪生成在 Boss 位置。  
若目标距离场中大于 `4m`，小怪生成在 Boss 到目标连线上，距目标 `4m` 的位置。

Boss 与小怪在 `5300ms` 后开始读条消灭之脚，读条 `4700ms`。  
读条开始时，各自锁定最近存活玩家方向：

- 未来终结：向锁定方向前方释放 `180°`、半径 `20m` AOE
- 过去终结：向锁定方向后方释放 `180°`、半径 `20m` AOE

消灭之脚只在判定前 `500ms` 显示 `500ms` 扇形预兆；范围仍在读条开始时锁定。

## 调试 Bot

调试 Bot 使用首次点名固定分出 A/B 组：

- A 组：`2` 分摊 + `1` 扇形 + `1` 大圈
- B 组：`2` 扇形 + `2` 大圈

组内优先级固定为 `MT -> ST -> H1 -> H2 -> D1 -> D2 -> D3 -> D4`。

处理顺序：

- A 组处理第 `1/2/3/8` 轮塔
- B 组处理第 `4/5/6/7` 轮塔

Bot 路线按当前轮点名组成分为带分摊组与无分摊组两类固定点。  
偶数轮消灭之脚前，8 个 Bot 统一移动到下一轮双塔角平分线同向或对向的 Boss 目标圈点引导。  
读条开始锁定方向后，Bot 立即离开引导点并移动到下一轮塔处理点。

偶数轮踩塔时，非处理组中负责引导扇形的两人站在各自塔外侧，不站在双塔之间。  
非处理组中负责引导近场点名的两人站在对应塔对向的 Boss 目标圈内侧 `1m`，即半径 `5m` 的位置。

第 `8` 轮消灭之脚读条开始锁定后，8 个 Bot 统一移动到同一个安全点躲避最后一轮半场刀。

## 失败文案

机制失败时调用 `state.fail(reason)`，不中断后续脚本，最终统一结算。

- 塔人数不足：`第 {round} 轮塔人数不足`
- 分摊人数错误：`分摊人数错误`
- 机制死亡：`{actor} 因 {sourceLabel} 死亡`
