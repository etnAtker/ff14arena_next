# 伊甸P4特殊：光与暗的龙诗

本文档描述《最终幻想 XIV》伊甸 P4 特殊练习内容“光与暗的龙诗”的模拟设计。

当前目标是做单一机制练习内容，不做完整副本复刻。  
本文档中的战斗规则实现于 `packages/content`，前端与服务端不得实现第二套战斗规则。

## 战斗范围

### 内容标识

- 战斗 id：`eden_p4_special`
- 战斗名称：`伊甸P4特殊`
- 首领：`伊甸`
- 小队：8 人，标准 `MT/ST/H1/H2/D1/D2/D3/D4`

### 练习边界

本战斗只包含以下内容：

- 光与暗的龙诗读条
- 光之锁点名与闭合连线
- 光之束缚连线长度检查
- 黑暗狂水点名
- 双人塔
- 光之波动扇形引导
- Bot 固定跑法
- 机制成功或失败结算

本战斗不包含以下内容：

- 伊甸 P4 完整时间轴
- 职业技能循环、治疗、减伤和复活
- 复杂 Bot 决策或重规划

## 场地与坐标

- 场地形状：圆形
- 场地中心：`(0, 0)`
- 场地半径：`20m`
- 首领初始位置：`(0, 0)`
- 首领目标环半径：`5m`

项目坐标按以下方向书写：

- 北：`(0, -r)`
- 东：`(r, 0)`
- 南：`(0, r)`
- 西：`(-r, 0)`

地图标点从正北开始顺时针均匀分布，半径 `10m`，顺序为 `A 2 B 3 C 4 D 1`。

## 时间轴

时间以战斗开始后毫秒计。

| 时间      | 事件                   | 数据                         |
| --------- | ---------------------- | ---------------------------- |
| `3000ms`  | 开始读条光与暗的龙诗   | 读条 `5000ms`                |
| `8000ms`  | 光与暗的龙诗读条完成   | 生成点名、连线、黑暗狂水、塔 |
| `8000ms`  | 开始读条光之波动       | 读条 `11000ms`               |
| `15000ms` | 光之锁转换为光之束缚   | 开始持续检查连线长度         |
| `18500ms` | 光之波动扇形预警出现   | `4` 个 `30°` 扇形            |
| `19000ms` | 光之波动结算与机制完成 | 成功或失败                   |

## 点名与状态

光之锁点名包含：

- `1` 名 T
- `1` 名 H
- `2` 名 DPS

光之锁玩家按随机闭合顺序生成 4 条展示连线。  
连线不会传递，只用于展示闭合关系和 Bot 跑法判断。

光之锁持续 `7000ms`。持续结束后转换为光之束缚。  
光之束缚期间，每条闭合连线长度必须保持在 `17m` 到 `23m` 之间。

黑暗狂水点名包含：

- `1` 名光之锁玩家
- `1` 名非光之锁玩家

光之波动结算时，两名黑暗狂水必须分别位于上下半场。

## 塔与扇形

塔：

- 北塔中心：`(0, -9)`
- 南塔中心：`(0, 9)`
- 塔半径：`3m`
- 每座塔必须正好由 `2` 名光之束缚玩家处理

扇形：

- 光之波动结算前 `500ms`，对距离场中最近的 `4` 名存活玩家显示扇形预警
- 每个扇形从场中朝该目标方向展开
- 扇形角度：`30°`
- 扇形半径：`20m`
- 光之束缚玩家被扇形命中时团灭
- 非光之束缚玩家同次光之波动内重复被扇形命中时死亡

DPS 排列：

- 光之波动结算时，按玩家相对场中的顺时针角度排序
- 4 名 DPS 必须连续出现，或 4 名 DPS 必须间隔出现

## Bot 跑法

Bot 当前是确定性脚本，不实现完整智能决策。  
Bot 只读取快照和 `scriptState`，再提交标准移动输入。

### 读条队列

玩家与 Bot 初始出生点即读条队列点。  
T/H 在北侧排队，DPS 在南侧排队。

| 槽位 | 坐标       |
| ---- | ---------- |
| MT   | `(-3, -5)` |
| ST   | `(-1, -5)` |
| H1   | `(1, -5)`  |
| H2   | `(3, -5)`  |
| D1   | `(-3, 5)`  |
| D2   | `(-1, 5)`  |
| D3   | `(1, 5)`   |
| D4   | `(3, 5)`   |

### 最终固定点

| 点位   | 坐标         |
| ------ | ------------ |
| 北塔左 | `(-0.6, -9)` |
| 北塔右 | `(0.6, -9)`  |
| 南塔左 | `(-0.6, 9)`  |
| 南塔右 | `(0.6, 9)`   |
| D 北   | `(-4, -1.2)` |
| D 南   | `(-4, 1.2)`  |
| B 北   | `(4, -1.2)`  |
| B 南   | `(4, 1.2)`   |

塔内点位于塔中心左右两侧。  
南北塔点之间的光之束缚距离必须保持在 `17m` 到 `23m` 之间。

### Bot 移动阶段

Bot 在光与暗的龙诗读条结束后不立即移动。  
读条结束后，Bot 延迟 `2000ms`，继续停留在读条队列点。

延迟结束后，Bot 分两段执行跑法：

1. 假设位置阶段：只执行换位 1，不执行换位 2，也不执行未换位 1 的黑暗狂水对角扇形交换。
2. 最终位置阶段：假设位置阶段持续 `3000ms` 后，再执行换位 2 或未换位 1 的黑暗狂水对角扇形交换。

### 换位 1 判定

检查 4 条闭合连线。  
若存在 T/H 组内部连线或 DPS 组内部连线，则需要换位 1。

换位 1 规则：

1. 找到被点名的 H。
2. 找到与该 H 相连的 DPS。
3. 该 H 与该 DPS 交换读条队列位置。

### 未换位 1

基础分配：

- 北塔：被点名 T/H，按北侧队列从左到右进入北塔左、北塔右
- 南塔：被点名 DPS，按南侧队列从左到右进入南塔左、南塔右
- 北侧扇形：未点名 T/H，按北侧队列从左到右去 D 北、B 北
- 南侧扇形：未点名 DPS，按南侧队列从左到右去 D 南、B 南

若两名黑暗狂水同半场，则执行未换位 1 专用交换：

- 有黑暗狂水的扇形玩家，与中心对角的扇形玩家交换
- 交换映射为 `D北 <-> B南`、`B北 <-> D南`

该交换用于同时满足黑暗狂水上下半场和 DPS 连续排列。

### 已换位 1

基础分配：

- 北塔左：被点名 T
- 北塔右：与 H 连线并换位的 DPS
- 南塔左：另一名被点名 DPS
- 南塔右：被点名 H
- 北侧未点名 2 人按换位后的队列从左到右先去 D 北、B 北
- 南侧未点名 2 人按换位后的队列从左到右先去 D 南、B 南
- D 方向两名引导者交换位置，即 `D北 <-> D南`

若两名黑暗狂水同半场，则执行换位 2：

- 北塔左 `<->` 北塔右
- 南塔左 `<->` 南塔右
- D 北 `<->` D 南
- B 北 `<->` B 南

## 失败文案

机制失败时调用 `state.fail(reason)`，不中断后续脚本，最终统一结算。

- 光之束缚玩家死亡：`光之束缚玩家死亡`
- 光之束缚连线目标缺失：`光之束缚连线目标缺失`
- 光之束缚连线长度错误：`光之束缚连线长度错误`
- 光之波动塔处理错误：`光之波动塔处理错误`
- 光之束缚玩家被扇形命中：`光之束缚玩家被扇形命中`
- 光之波动扇形易伤：`{actor} 因 光之波动扇形易伤 死亡`
- 黑暗狂水未分处上下半场：`黑暗狂水未分处上下半场`
- DPS 站位顺序错误：`DPS 站位顺序错误`
