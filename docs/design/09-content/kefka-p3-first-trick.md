# 凯夫卡P3：一运

本文档描述《最终幻想 XIV》凯夫卡 P3 一运练习内容的模拟设计。

本战斗规则实现于 `packages/content`，前端与服务端不得实现第二套战斗规则。  
当前版本只实现机制判定，不实现 Bot 跑法。

## 战斗范围

- 战斗 id：`kefka_p3_first_trick`
- 战斗名称：`凯夫卡P3：一运`
- 首领：`凯夫卡`
- 小队：8 人，标准 `MT/ST/H1/H2/D1/D2/D3/D4`
- 场地半径：`20m`
- 首领目标圈：隐藏

## 暴雷

`16000ms` 开始读条暴雷，并在读条开始时锁定暴雷中心。

暴雷中心按 MT 当时位置计算：

- 若 MT 到场中距离小于等于 `4m`，暴雷中心为场中 `(0,0)`
- 否则取场中 `(0,0)` 指向 MT 的射线上、距离 MT `4m` 且更靠近场中的点

锁定位置同步生成表现用 NPC `艾克斯德司`。
该 NPC 显示为橙色敌人标记，并带有红色 `4m` 目标圈。
该 NPC 只作为视觉表现，不参与碰撞、仇恨、目标选择或伤害来源逻辑。
暴雷读条期间不显示伤害范围。

`23000ms` 暴雷判定：

- 以锁定位置为中心显示白色半径 `11m` 范围
- 范围内玩家获得永久 `死亡` buff，并记录失败原因
- 失败来源：`被暴雷命中`

`27000ms` 开始第二组暴雷读条，读条名称仍为 `暴雷`，读条 `6000ms`。

`33000ms` 与 `36000ms` 各判定一次：

- 每次判定时重新选择距离 `艾克斯德司` 最近的存活玩家
- 以该玩家当前位置为中心短暂显示白色半径 `5m` 范围
- 范围内 `MT/ST` 受到普通机制伤害
- 范围内非 `MT/ST` 玩家获得永久 `死亡` buff，并记录失败来源 `被暴雷命中`

## 经度聚爆 / 纬度聚爆

`3000ms` 在场中生成表现用 NPC `卡奥斯`，显示为敌人标记，并带红色 `6m` 目标圈。

`4000ms` 到 `40000ms` 之间，`卡奥斯` 每秒按 ST 当前位置重算一次位置。每次重定位时，将当前 `卡奥斯` 位置记为 `C`，将当时 ST 位置记为 `S`：

- 若 `C` 到 `S` 距离小于等于 `6m`，`卡奥斯` 保持原位置
- 否则 `卡奥斯` 移动到 `S + normalize(C - S) * 6`，即保留相对 ST 的方向并距离 ST `6m`

每秒移动通过结束旧 `卡奥斯` 标记并在新位置生成同名标记表现；不新增 marker 动画能力。

`41000ms` 随机开始读条 `经度聚爆` 或 `纬度聚爆`，读条 `5000ms`。

读条开始时，不重置 `卡奥斯` 到场中；直接读取当前 `卡奥斯` 位置作为聚爆中心，并锁定卡奥斯面向为该位置指向当时 ST 的方向。
该位置和面向只写入脚本状态用于扇形判定，不扩展 UI 标记朝向；读条开始后不再随 ST 移动改变。`卡奥斯` 标记持续存在到 `67000ms` 超级跳判定。

`46000ms` 与 `48000ms` 各判定一次：

- `经度聚爆`：先判定卡奥斯前后方向，再判定左右方向
- `纬度聚爆`：先判定卡奥斯左右方向，再判定前后方向
- 每次判定以锁定的 `卡奥斯` 位置为中心生成两片相对的 `90°`、`60m` 扇形范围
- 命中玩家获得永久 `死亡` buff，并记录失败来源 `被扇形命中`

## 超级跳

`61000ms` 锁定超级跳位置。

- 从当前存活玩家中选择距离场中心 `(0,0)` 最远者
- 若距离相同，按 `PARTY_SLOT_ORDER` 选择靠前玩家
- 锁定该玩家当时位置，之后玩家移动不改变结算点
- 带 `死亡` buff 但仍 `alive` 的玩家继续参与锁定选择

`67000ms` 超级跳判定：

- `卡奥斯` 瞬移到锁定位置，并在该位置重新生成同名敌人标记和红色 `6m` 目标圈，持续到 `86750ms`
- 以锁定位置为中心短暂显示半径 `11m` 的 `超级跳范围`
- 范围内玩家获得永久 `死亡` buff，并记录失败来源 `被超级跳命中`
- 不触发击退、不消耗风类状态、不排队风分摊追击

## 真空波

`37000ms` 到 `60000ms` 之间，每秒重新定位一次 `艾克斯德司`。
每次重定位时，将当前 `艾克斯德司` 位置记为 `E`，将当时 MT 位置记为 `M`：

- 若 `E` 到 `M` 距离小于等于 `4m`，`艾克斯德司` 保持原位置
- 否则 `艾克斯德司` 移动到 `M + normalize(E - M) * 4`，即保留相对 MT 的方向并距离 MT `4m`

每秒移动通过结束旧 `艾克斯德司` 标记并在新位置生成同名标记表现；不新增 marker 动画能力。

`61000ms` 开始读条 `真空波`，读条 `8000ms`。

读条开始时，`艾克斯德司` 再按同一规则进行一次最终定位：

- 若 `E` 到 `M` 距离小于等于 `4m`，`艾克斯德司` 保持原位置
- 否则 `艾克斯德司` 移动到 `M + normalize(E - M) * 4`，即保留相对 MT 的方向并距离 MT `4m`

`61000ms` 的最终定位后，新的 `艾克斯德司` 标记持续到机制结束。读条开始后位置不再随 MT 移动变化。

`69000ms` 真空波判定：

- 不额外显示范围
- 以锁定后的 `艾克斯德司` 为击退源，对全场存活玩家判定击退
- 无风类状态时击退 `20m`
- 风类状态处理正确时击退 `10m`，处理失败时击退 `40m`
- 未开启防击退且持有风类状态的玩家，击退后移除该状态，并累计一次风分摊追击
- 开启防击退的玩家不被击退，也不移除风类状态，不累计风分摊追击

真空波触发的风分摊追击仍在状态移除后延迟 `1500ms` 判定。

## 冲锋 / 麻将

`54000ms` 开始冲锋点表现：

- 随机选择基准方向：上、右、下、左之一
- 同时随机选择旋转方向：顺时针或逆时针，本次机制内保持不变
- 在基准方向距场中 `16m` 的位置生成小紫色圆形 `冲锋点`

`62000ms` 起，`冲锋点` 移动到基准方向距场中 `25m` 的场外位置。
之后每 `2000ms` 瞬移到按固定旋转方向旋转 `45°` 后的位置，共旋转 `7` 次。
最后一次位置在 `76000ms` 出现，持续 `2000ms`，`78000ms` 消失。

`72000ms` 随机给 8 名玩家分配唯一顺位 `1-8`：

- 顺位显示在玩家头顶，持续到 `80000ms`
- 奇数顺位显示蓝色
- 偶数顺位显示红色

`84000ms` 到 `85750ms` 每 `250ms` 判定一次麻将矩形，共 8 次：

- 第 1 次从基准方向 `25m` 场外点出发，之后按冲锋点旋转顺序依次使用后续 7 个场外起点
- 第 `n` 次朝顺位 `n` 玩家判定当刻的当前位置发射矩形
- 矩形从起点向目标方向延伸，长度 `50m`，宽度 `12m`
- 矩形判定时短暂显示约 `1000ms`
- 顺位本人若距离起点小于 `40m`，获得永久 `死亡` buff，并记录失败来源 `麻将距离过近`
- 其它玩家若被该矩形命中，获得永久 `死亡` buff，并记录失败来源 `麻将被其它人的矩形命中`
- 顺位本人距离足够时，不因自己的矩形覆盖而获得死亡 buff

## 元素块

深层痛楚读条完成后，在 `(|x|=10, |y|=10)` 的四个角落中随机选择三个连续角落生成元素块。

- 水元素块：蓝色方形
- 火元素块：红色三角
- 风元素块：绿色菱形

风元素块固定在火元素块与水元素块中间。  
元素块只作为机制目标和击退源，不作为碰撞体。

## 状态赋予

深层痛楚读条完成后，所有玩家获得 `68000ms` 的风类状态。  
风类状态按随机顺序拆成 `4` 名混沌之风与 `4` 名混沌之逆风。

- 混沌之风：状态 id `1602`
- 混沌之逆风：状态 id `1603`

同时赋予炎水状态：

- `1` 名 T/H 与 `1` 名 DPS 获得混沌之炎，状态 id `1600`
- 另 `1` 名 T/H 与 `1` 名 DPS 获得混沌之水，状态 id `1601`
- 炎与水随机决定一个持续 `19000ms`，另一个持续 `46000ms`

## 炎水到期

所有机制伤害命中时，若目标已有 `injury_up`，目标获得永久 `死亡` buff，并记录失败原因。
若目标没有 `injury_up`，受到伤害并获得 `3000ms` 易伤。
`死亡` buff 仅作为测试标记，不会让玩家进入真实死亡状态，也不会阻止玩家继续移动、被点名或参与后续机制。

混沌之炎到期时：

- 移除混沌之炎
- `500ms` 后以自身为中心结算半径 `5m` AOE，附加易伤
- 累计一次火元素追击

混沌之水到期时：

- 移除混沌之水
- `500ms` 后以自身为中心结算内径 `5m`、外径 `10m` 环形 AOE，附加易伤
- 累计一次水元素追击

## 元素追击

炎、水到期累计的追击不会立即判定。  
脚本在累计后 `1500ms` 统一处理同类追击。

若同一时点累计了 `2` 次火元素追击，则在判定时选中距离火元素块最近的 `2` 名不同存活玩家，分别以他们为中心生成追击环形 AOE。  
不会对同一个最近玩家重复释放 `2` 次。

火元素追击：

- 目标：距离火元素块最近的不同玩家
- 范围：以内径 `5m`、外径 `10m` 的环形 AOE 结算
- 命中：附加易伤，并按火元素块为击退源击退

水元素追击：

- 目标：距离水元素块最近的不同玩家
- 范围：半径 `5m` 圆形 AOE
- 命中：附加易伤，并按水元素块为击退源击退

## 风类状态与击退

元素追击命中玩家时，根据玩家风类状态与面向调整击退距离。

基础击退距离为 `20m`。

混沌之风：

- 面对击退源：击退 `40m`
- 背对击退源：击退 `10m`

混沌之逆风：

- 面对击退源：击退 `10m`
- 背对击退源：击退 `40m`

面对/背对按玩家当前面向与“玩家指向击退源”的夹角判定，夹角小于等于 `90°` 视为面对，否则视为背对。

持有混沌之风或混沌之逆风的玩家吃到元素追击击退时，脚本移除该风类状态，并累计一次风分摊追击。

## 风分摊追击

风类状态被元素追击消除后，不会立即判定分摊。  
脚本在累计后 `1500ms` 统一处理风分摊追击。

每次风分摊追击选中距离水元素块最近的 `1` 名不同存活玩家，释放半径 `6m` 的 `2` 人分摊 AOE。

- 范围内少于 `2` 人时，记录失败并使目标获得永久 `死亡` buff
- 范围内至少 `2` 人时，范围内玩家受到机制伤害并附加易伤

## 失败文案

机制失败时调用 `state.fail(reason)`，不中断后续脚本，最终统一结算。

- 风分摊人数不足：`混沌之风分摊人数不足`
- 暴雷命中：`被暴雷命中`
- 聚爆扇形命中：`被扇形命中`
- 超级跳命中：`被超级跳命中`
- 麻将距离过近：`麻将距离过近`
- 麻将被其它人的矩形命中：`麻将被其它人的矩形命中`
- 死亡 buff 失败：`{actor} 因 {sourceLabel} 死亡`
