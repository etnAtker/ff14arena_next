# 凯夫卡P5：癫狂交响曲

本文档描述《最终幻想 XIV》凯夫卡 P5 癫狂交响曲练习内容的模拟设计。

本战斗规则实现于 `packages/content`，前端与服务端不得实现第二套战斗规则。  
当前版本实现机制判定和固定路线 Bot 跑法。

## 战斗范围

- 战斗 id：`kefka_p5_mad_symphony`
- 战斗名称：`凯夫卡P5：癫狂交响曲`
- 首领：`凯夫卡`
- 小队：8 人，标准 `MT/ST/H1/H2/D1/D2/D3/D4`
- 场地半径：`20m`
- 首领目标圈：`8m`
- 场地标点：沿用凯夫卡 P3 二运 `A 2 B 3 C 4 D 1` 八方标点，标点距离场中 `12m`

## 时间轴

时间以开战瞬间起算。

| 时间      | 事件           | 数据                                                                         |
| --------- | -------------- | ---------------------------------------------------------------------------- |
| `0ms`     | 癫狂交响曲读条 | 凯夫卡读条 `5000ms`                                                          |
| `5200ms`  | 第一轮预兆     | `MT/ST` 红色 `5m` 死刑，`DPS` 随机 `3` 人蓝色 `5m` 分散                      |
| `5700ms`  | 第一轮判定     | 第一轮死刑和蓝色分散结算，并给双 T 随机赋予核爆、神圣 Buff                   |
| `8200ms`  | 第二轮预兆     | `MT` 位置红色 `5m` 双 T 分摊，当前位置最近的 `3` 名非 T 蓝色 `5m` 分散       |
| `8700ms`  | 第二轮判定     | 第二轮双 T 分摊和蓝色分散结算                                                |
| `11700ms` | 核爆与神圣判定 | 按双 T Buff 持有者结算                                                       |
| `14700ms` | 后续第一轮判定 | 重新随机 `1T + 1H + 1DPS`，显示 `T/H/D` 三组 `5m` 职能分摊，范围保留 `300ms` |
| `16700ms` | 后续第二轮判定 | 再次重新随机 `1T + 1H + 1DPS`，规则同后续第一轮                              |
| `17700ms` | 战斗完成       | 结束本练习内容                                                               |

`DH` 表示 `H1/H2/D1/D2/D3/D4`。  
前两轮分散和死刑预兆提前 `500ms` 显示；后续两轮职能分摊在判定同 tick 显示，并保留 `300ms`。

## 第一轮判定

`MT/ST` 各自获得半径 `5m` 红色死刑范围：

- 命中 `MT/ST` 时造成机制伤害，并赋予 `1000ms` 易伤。
- 非 `MT/ST` 命中时立即死亡。
- 已有易伤的玩家再次受到机制伤害时立即死亡。

## Bot 跑法

Bot 初始站位为 Boss 目标圈外 `1m`，即场中半径 `9m`。  
从正北开始顺时针每 `45°` 依次为：

`MT -> ST -> H2 -> D2 -> D4 -> D1 -> H1 -> D3`

第一轮判定前，Bot 保持初始八方分散站位。  
第一轮判定后，`ST` 移动到 `MT` 位置处理第二轮双 T 分摊；第一轮蓝圈 DPS 外移，避免成为第二轮最近非 T。

第二轮判定后，Bot 按第一轮 Buff 结果移动：

- “顺手加个核爆”持有者移动到正北 A 方向场边内侧，目标半径 `19m`。
- “顺手加个神圣”持有者与第一轮蓝圈 `3` 名 DPS 移动到正南 Boss 目标圈上集合分摊，目标半径 `8m`。
- 第二轮蓝圈与其他非分摊玩家避开核爆和神圣范围。

Buff 判定后，Bot 按职能移动到后续分摊点：

- `MT/ST` 到正北 Boss 目标圈上。
- `H1/H2` 到左下方 Boss 目标圈上。
- `D1/D2/D3/D4` 到右下方 Boss 目标圈上。

`D1/D2/D3/D4` 中随机 `3` 人获得半径 `5m` 蓝色分散范围：

- 命中玩家受到机制伤害，并赋予 `4000ms` 易伤。
- 已有易伤的玩家再次受到机制伤害时立即死亡。

`MT/ST` 随机分配 `6000ms` “顺手加个核爆”和 `6000ms` “顺手加个神圣”，两种 Buff 不会给到同一名玩家。

状态展示使用 XIVAPI 状态：

- “顺手加个核爆”：状态 `5350`，图标 `214408`
- “顺手加个神圣”：状态 `5351`，图标 `214409`

## 第二轮判定

`8700ms` 以 `MT` 当前位置为中心结算半径 `5m` 双 T 分摊：

- 范围内必须有 `MT/ST` 两名坦克。
- 非 T 命中时立即死亡。
- 双 T 分摊成功时，范围内坦克受到机制伤害并赋予 `1000ms` 易伤。
- 少于两名坦克时记录失败原因“癫狂交响曲双T分摊人数不足”，并击杀范围内坦克。

第二轮蓝色分散锁定预兆时当前位置最近的 `3` 名非 T，规则与第一轮蓝色分散相同。

## 核爆与神圣

`11700ms` 同时判定第一轮赋予的两个 Buff。脚本使用第一轮记录的 Buff 持有者判定，不依赖同 tick 状态是否仍在状态栏中。

“顺手加个神圣”：

- 以持有者当前位置为中心，半径 `5m`。
- 判定时范围内至少 `4` 名机制有效玩家才算成功。
- 成功时范围内玩家受到机制伤害，并赋予 `3000ms` 易伤。
- 少于 `4` 人时记录失败原因“顺手加个神圣分摊人数不足”，并击杀范围内玩家。

“顺手加个核爆”：

- 以持有者当前位置为中心，半径 `25m`。
- 命中玩家受到机制伤害，并赋予 `3000ms` 易伤。
- 已有易伤的玩家再次受到机制伤害时立即死亡。

## 后续职能分摊

`14700ms` 和 `16700ms` 各结算一轮后续职能分摊。  
每轮都会重新随机 `1` 名坦克、`1` 名治疗和 `1` 名 DPS。

- 坦克目标显示 `T` 分摊，半径 `5m`，需要至少 `2` 名机制有效玩家。
- 治疗目标显示 `H` 分摊，半径 `5m`，需要至少 `2` 名机制有效玩家。
- DPS 目标显示 `D` 分摊，半径 `5m`，需要至少 `4` 名机制有效玩家。
- 成功时范围内玩家受到机制伤害，并赋予 `1000ms` 易伤。
- 人数不足时记录对应失败原因，并击杀范围内玩家。
- 已有易伤的玩家再次受到机制伤害时立即死亡。

Bot 后续职能分摊固定为：`T` 组到正北 Boss 目标圈上，`H` 组到左下方 Boss 目标圈上，`D` 组到右下方 Boss 目标圈上。
