# Web 客户端设计

## 技术栈

- Vue 3
- Naive UI
- PixiJS
- Pinia
- Socket.IO Client
- TypeScript

## 职责划分

Vue 负责页面结构与状态编排。  
Naive UI 负责表单、按钮、卡片、列表、状态标签和信息面板等通用 UI 组件。  
PixiJS 负责场地、角色、Boss、AOE 和标记绘制。  
Pinia 负责房间状态、当前用户状态、当前战斗元数据和 UI 状态。  
业务规则不写入前端 Store。

## 页面结构

客户端包含两个页面：

- 首页与房间列表页
- 模拟页

### 首页与房间列表页

页面结构如下：

- 顶部标题栏
- 房间列表区
- 创建房间弹窗
- 加入房间弹窗

创建房间弹窗包含：

- 房间名称输入框
- 战斗下拉选择框
- 创建按钮

### 模拟页

进入房间后立即进入模拟页。  
等待态与战斗中都使用同一页面，不再单独显示房间大厅页和结算页。

页面采用三栏布局：

- 左侧人员栏
- 中间绘制区域
- 右侧结果与日志栏

左侧人员栏固定显示 8 个槽位：

- `MT`
- `ST`
- `H1`
- `H2`
- `D1`
- `D2`
- `D3`
- `D4`

每个槽位块分两行：

- 第一行显示 `[槽位] 名字` 与操作按钮
- 第二行显示 `当前HP / 最大HP`

视觉规则如下：

- 坦克槽位使用蓝色底色
- 治疗槽位使用绿色底色
- DPS 槽位使用红色底色
- 当前自己额外使用紫色强调

按钮规则如下：

- 自己的槽位：普通玩家为准备，房主为开始
- 其他槽位：切换
- 切换槽位只允许在 `waiting` 阶段进行

中间绘制区域内容：

- 由当前战斗定义的场地边界
- Boss 目标环
- 玩家与 Bot 位置点
- 玩家朝向箭头
- AOE 预警图形
- 分摊 / 分散圈

Boss 读条绘制在场地区域顶部，显示：

- 当前技能名
- 进度条

角色图标使用职能色并在中心显示简写：

- `MT -> M`
- `ST -> S`
- `H1 -> 1`
- `H2 -> 2`
- `D1 -> 1`
- `D2 -> 2`
- `D3 -> 3`
- `D4 -> 4`

右侧结果与日志栏显示：

- 上一轮结果
- 失败原因列表
- 当前 8 人状态摘要
- 实时战斗日志

主动按钮只保留：

- 防击退按钮
- 重置缩放按钮

## 操控设计

客户端提供两种可选操控模式：

- 传统
- 标准

操控模式在个人设置面板中切换。  
该设置保存在客户端本地。

### 传统模式

规则如下：

- 左键拖动改变相机朝向
- 右键拖动改变相机朝向
- 左键和右键拖动都不立即改变玩家面向
- `WASD` 负责移动
- 移动方向以当前相机方向为准
- 玩家面向在移动时变为移动方向
- 移动时不改变相机朝向

### 标准模式

规则如下：

- 左键拖动只改变相机朝向
- 右键拖动同时改变相机朝向和人物朝向
- `WASD` 负责移动
- 移动方向以人物当前面向为准
- 移动时不改变人物朝向
- 移动时不改变相机朝向

### 通用规则

两种模式共同遵守以下规则：

- 键盘 `1` 触发防击退
- 移动输入按 Tick 连续发送
- 朝向输入在状态变更时发送

## HUD 接口

- `bossCastBar`
- `scriptState` 只作为脚本 / Bot controller 使用，不参与前端展示

## 镜头规则

模拟页镜头规则如下：

- 镜头朝向由鼠标拖拽控制
- 当前受控玩家始终绘制在画面中心点偏下固定位置
- 鼠标滚轮控制缩放
- 双击空白区域重置缩放
- 当没有可控角色时，画面回到当前战斗场地中心绘制
