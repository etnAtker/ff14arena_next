# Web 客户端设计

## 技术栈

- Vue 3
- PixiJS
- Pinia
- Socket.IO Client
- TypeScript

## 职责划分

Vue 负责页面结构、表单、按钮和 HUD 容器。  
PixiJS 负责场地、角色、Boss、AOE 和标记绘制。  
Pinia 负责房间状态、当前用户状态、当前战斗元数据和 UI 状态。  
业务规则不写入前端 Store。

## 页面结构

客户端包含三个页面：

- 首页与房间大厅页
- 战斗页
- 结算页

### 首页与房间大厅页

页面结构如下：

- 顶部标题栏
- 房间列表区
- 创建房间弹窗
- 加入房间弹窗

创建房间弹窗包含：

- 房间名称输入框
- 战斗下拉选择框
- 创建按钮

房间大厅页包含：

- 左侧 8 槽位面板
- 中部战斗摘要卡片
- 右侧聊天与系统消息面板
- 底部准备按钮与开始按钮

8 槽位面板固定展示：

- `MT`
- `ST`
- `H1`
- `H2`
- `D1`
- `D2`
- `D3`
- `D4`

每个槽位卡片展示：

- 玩家名或 Bot 名
- 在线状态
- 准备状态
- 当前血量
- 是否持有防击退

房主操作：

- 选择战斗
- 踢出玩家
- 将空槽补成 Bot
- 将 Bot 移除为空槽
- 调整槽位
- 点击开始

普通玩家操作：

- 选择空槽加入
- 点击准备
- 退出房间

### 战斗页

页面结构如下：

- 主场地画布
- 队伍状态面板
- Boss 读条区
- 战斗信息条
- 技能与状态条
- 日志区
- 重开按钮
- 返回大厅按钮

主场地画布内容：

- 由当前战斗定义的场地边界
- Boss 目标环
- 玩家位置点
- 玩家朝向箭头
- AOE 预警图形
- 点名标记
- 分摊圈
- 分散圈
- 伤害飘字

队伍状态面板固定显示 8 名成员：

- 槽位名
- 当前 HP
- 存活状态
- 受伤加重状态
- 防击退状态
- 防击退冷却

Boss 读条区显示：

- 当前技能名
- 总读条时间
- 当前进度

战斗信息条显示：

- 当前战斗名
- 战斗已进行时间
- 最近一次战斗提示文本
- 当前失败原因聚合显示

技能与状态条只保留一个主动按钮：

- 防击退按钮
- 冷却数字
- 剩余持续时间

### 结算页

结算页显示：

- 本次战斗最终结果
- 全队最终存活情况
- 聚合后的失败原因列表
- 每名成员死亡原因
- 房主重开按钮
- 返回大厅按钮

## 操控设计

客户端提供两种可选操控模式：

- 传统
- 标准

操控模式在个人设置面板中切换。  
该设置保存在客户端本地。

### 传统模式

规则如下：

- 左键拖动改变相机朝向
- 右键拖动改变相机朝向
- 左键和右键拖动都不立即改变玩家面向
- `WASD` 负责移动
- 移动方向以当前相机方向为准
- 玩家面向在移动时变为移动方向
- 移动时不改变相机朝向

### 标准模式

规则如下：

- 左键拖动只改变相机朝向
- 右键拖动同时改变相机朝向和人物朝向
- `WASD` 负责移动
- 移动方向以人物当前面向为准
- 移动时不改变人物朝向
- 移动时不改变相机朝向

### 通用规则

两种模式共同遵守以下规则：

- 施放有目标技能时，人物面向朝向目标
- 防击退为无目标技能，不改变人物面向
- 键盘 `1` 触发防击退
- 移动输入按 Tick 连续发送
- 朝向输入在状态变更时发送

## HUD 接口

客户端固定提供以下 HUD 插槽：

- `bossCastBar`
- `battleMessage`
- `recentFailureReason`
- `centerHint`
- `countdownText`

战斗定义通过接口写入显示内容。  
前端不硬编码具体战斗文案。

当客户端收到失败结果标记时：

- 战斗页继续保持运行
- HUD 追加并重绘失败原因列表
- 不弹出结算页

只有收到 `encounterCompleted` 后，前端才切换到结算页。

## 镜头规则

战斗页镜头规则如下：

- 默认镜头居中于当前战斗场地中心
- 鼠标滚轮控制缩放
- 双击空白区域重置缩放
- 镜头不跟随单个玩家
