# MVP 范围

## MVP 目标

MVP 要证明以下产品链路成立：

- 房主能开房
- 玩家能加入
- 空位能补 Bot
- 能选择一个战斗
- 能多人开练
- 战斗按脚本稳定执行到结尾
- 玩家会因为吃错机制而掉血、死亡，并写入失败结果标记
- 房间能快速重开

## MVP 必须具备

- 房间系统
- 8 槽位系统
- Bot 自动补位
- 一个标准 2D 场地
- 一个 Boss
- 一个战斗脚本
- 圆形 / 环形 / 分摊 / 分散 helper
- 击退与防击退
- 简化血量与死亡
- 重开

## MVP 不实现

- 多职业循环
- 多副本
- 完整回放播放器
- 智能 Bot
- 复杂持久化
