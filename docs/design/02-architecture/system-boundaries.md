# 系统边界

## 总体边界

系统分为三层：

1. `core`
2. 服务端运行层
3. Web 客户端展示层

## core 边界

`core` 只处理领域逻辑。

输入：

- 初始化配置
- 玩家输入事件
- 战斗脚本
- Tick 推进

输出：

- 世界快照
- 增量事件
- 失败原因
- 结算状态

禁止依赖：

- DOM API
- Canvas / Pixi API
- Socket.IO API
- Fastify Request / Reply
- 浏览器本地存储

## 服务端边界

服务端负责：

- 管房
- 托管权威模拟
- 接收玩家输入
- 驱动 Bot 输入
- 广播结果
- 处理重连和同步

## 客户端边界

客户端负责：

- 收集输入
- 呈现状态
- 承载本地镜像
- 承载 UI 和绘制层

## 内容边界

战斗脚本是内容，不是引擎。

- “如何调度和结算”属于 `core`
- “某个 Boss 在第几秒做什么”属于 `content`
