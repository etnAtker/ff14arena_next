# 空间判定与效果结算

## 空间系统

空间系统使用二维平面，支持：

- 点
- 方向
- 圆形
- 环形
- 扇形
- 矩形

## 运动模型

运动模型采用简化实现：

- 位置连续
- 速度固定或由脚本指定
- 不实现角色互相阻挡
- 位移完成后执行边界判定

## 边界规则

Actor 超出场地边界后立即死亡。

## 强制位移

当前实现支持击退。

击退结算流程如下：

1. 检查目标是否存活
2. 检查目标是否具有防击退状态
3. 具有防击退状态时，仅记录抵抗事件
4. 否则按指定方向或径向移动
5. 位移完成后进行边界判定

## AOE 生命周期

AOE 生命周期固定为：

1. 生成
2. 预警显示
3. 结算
4. 销毁

## 伤害与效果顺序

默认顺序如下：

1. 判定命中
2. 结算伤害
3. 结算附加状态
4. 结算位移
5. 检查死亡
