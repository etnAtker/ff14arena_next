# 房间生命周期

## 房间阶段

房间生命周期如下：

1. `created`
2. `lobby`
3. `loading`
4. `running`
5. `finished`
6. `closed`

## created

该阶段完成：

- 分配房间 ID
- 建立房主
- 初始化默认槽位

## lobby

该阶段允许：

- 选择战斗
- 调整槽位
- 邀请玩家
- 加入和离开
- 填充或移除 Bot

## loading

该阶段负责：

- 加载战斗内容
- 初始化权威模拟
- 生成初始快照

## running

该阶段允许：

- 玩家输入移动、转向、防击退
- 房主请求重开

该阶段禁止：

- 切换战斗
- 改变槽位占用

## finished

该阶段表示本轮战斗已结束，但房间保留。  
结束原因包括：

- 战斗脚本自然结束并给出成功结果
- 战斗脚本自然结束并给出失败结果
- 房主中止

## closed

该阶段表示房间彻底关闭。

## 模拟实例关系

一个房间在 `running` 阶段只对应一个权威模拟实例。  
每次重开都创建新的模拟实例。
