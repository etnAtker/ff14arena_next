# 房间生命周期

## 房间阶段

房间生命周期如下：

1. `waiting`
2. `running`
3. `closed`

## waiting

该阶段表示房间处于待开始状态。  
客户端在进入房间后立即显示模拟页，但机制时间轴不推进。

该阶段允许：

- 选择战斗
- 切换槽位
- 玩家准备
- 房主开始
- 玩家移动与转向同步

该阶段规则如下：

- 房主固定视为已准备
- 其他玩家切槽、切战斗、战斗结束后重置为未准备
- 切换战斗后立即重建等待态场景
- 槽位始终补满为 8 人，不足位置由 Bot 占位

## running

该阶段表示房间正在推进权威模拟。

该阶段允许：

- 玩家输入移动、转向、防击退

该阶段禁止：

- 切换战斗
- 切换槽位
- 新玩家加入

当战斗脚本完成后：

- 房间立即回到 `waiting`
- 保留上一轮结果
- 不跳转独立结算页
- 直到下一次开始或切换战斗时才整体重置

## closed

该阶段表示房间被销毁。

触发条件如下：

- 房主主动离开
- 房主断开 WebSocket 连接

关闭后效果如下：

- 其他玩家被移出房间
- 前端返回大厅页

## 离房与补位规则

- 非房主在 `waiting` 阶段离开或断线：立即由 Bot 补位
- 非房主在 `running` 阶段离开或断线：本轮先保留原角色，回到 `waiting` 时再补 Bot
- 当前已死亡的角色在任何阶段都不能移动

## 模拟实例关系

- `running` 阶段只存在一个权威模拟实例
- `waiting` 阶段使用等待态快照，不推进战斗时间轴
- 每次点击开始都会基于当前战斗静态数据重新创建新的权威模拟实例
