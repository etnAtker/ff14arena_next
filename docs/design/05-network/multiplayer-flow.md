# 多人联机流程

## 设计前提

项目以多人联机为基础设计。  
房间人数不足时，由 Bot 自动补位。

## 房间流程

房间流程如下：

1. 房主创建房间
2. 房主选择战斗
3. 系统生成 8 个固定槽位
4. 玩家加入并占据槽位
5. 空槽由 Bot 填充
6. 房主点击开始
7. 服务端加载战斗并启动权威模拟
8. 房间进入战斗中
9. 战斗结束或房主重开

## 房间状态

房间状态包括：

- `lobby`
- `loading`
- `running`
- `finished`

## 槽位占用规则

槽位占用规则如下：

- 房主可以手动调整玩家与 Bot 的槽位
- 未手动调整时，系统按加入顺序填充空槽
- 开始时所有空位自动生成 Bot

## 输入流

客户端输入只发送意图：

- 目标移动点
- 改变朝向
- 使用防击退
- 准备就绪
- 请求重开

## 广播流

服务端广播：

- 房间状态变化
- 槽位变化
- 战斗开始与结束
- 失败结果标记
- 机制增量事件
- 周期性快照

## 重开

重开功能包含：

- 房主一键重开
- 维持当前槽位与 Bot 配置
- 保持当前战斗不变
- 创建新的模拟实例
