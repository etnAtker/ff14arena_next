# 多人联机流程

## 设计前提

项目以多人联机为基础设计。  
房间人数不足时，由 Bot 自动补位。

## 房间流程

房间流程如下：

1. 房主创建房间
2. 房主选择战斗
3. 系统生成 8 个固定槽位
4. 玩家加入并占据槽位
5. 空槽由 Bot 填充
6. 房主点击开始
7. 服务端加载战斗并启动权威模拟
8. 房间进入战斗中
9. 战斗结束后返回待开始

## 房间状态

房间状态包括：

- `waiting`
- `running`
- `closed`

## 槽位占用规则

槽位占用规则如下：

- 房间始终维持 8 个槽位
- 玩家可以在 `waiting` 阶段与 Bot 或其他玩家互换槽位
- 非房主离开后由 Bot 补位
- 房主离开后房间立即销毁

## 输入流

客户端输入只发送意图：

- 目标移动点
- 改变朝向
- 使用防击退
- 准备就绪

## 广播流

服务端广播：

- 房间状态变化
- 槽位变化
- 战斗开始与结束
- 失败结果标记
- 机制增量事件
- 周期性快照

## 再次开始

再次开始功能包含：

- 房主在 `waiting` 阶段再次点击开始
- 保持当前槽位与战斗选择
- 清空上一轮结果与日志
- 创建新的模拟实例并重置所有角色状态
