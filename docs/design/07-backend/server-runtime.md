# 服务端运行时设计

## 技术栈

- Fastify
- Socket.IO
- TypeScript

## 核心职责

服务端负责：

- 房间管理
- 玩家连接管理
- 权威模拟运行
- Bot 运行
- 状态同步

服务端在战斗运行期间维护结果标记状态。  
失败结果标记写入后，服务端继续推进模拟，直到战斗脚本结束。  
服务端同时维护失败原因集合，并在每次新增原因时向客户端广播增量事件。

## 房间实例

每个房间实例维护：

- 房间元数据
- 槽位状态
- 当前战斗
- 当前模拟实例
- 当前 Bot 控制器集合
- 最近快照缓存

## 权威模拟循环

权威模拟循环实现如下：

- Node 定时器驱动
- 固定 Tick 间隔
- 每 Tick 批量处理输入

## Bot 托管

Bot 控制器与权威模拟同进程运行。

## 断线与重连

断线与重连规则如下：

- 玩家断线后槽位暂时保留
- 重连后恢复原槽位
- 长时间未归时，房主可以将该槽位改为 Bot

## 存储

第一阶段存储覆盖以下内容：

- 用户基础身份
- 房间基础信息
- 战斗元数据

回放和统计只保留接口位置。
