## 5. Vue 高频刷新优化

目标是减少不必要的 Vue 响应式高频写入，把逐帧绘制和纯预览状态尽量留在 Pixi 或非响应式状态中。

已完成：

- 降低 HUD 显示时钟刷新频率
  - `BattlePage` 的 HUD 文本时钟已改为 `100ms`
  - 状态倒计时和技能 CD 文案不再依赖 Vue 每帧刷新
- 保持 Boss 读条连续
  - Boss 读条进度已改为 CSS 线性动画
  - Vue 只在读条身份变化时建立动画锚点
  - 同一读条期间，事件或快照时间变化不会重置 CSS 动画时间轴
- 合并滚轮缩放更新
  - `BattleStage` 已使用 RAF 合并滚轮事件
  - 每帧最多同步一次 `cameraZoom`

剩余建议优化点：

- 减少拖拽镜头时的 Vue 更新
  - 当前拖拽镜头时 `cameraYaw` 会按 RAF 写入 Vue ref
  - 可评估把拖拽中的镜头角度放在 `BattleStage` 内部非响应式变量，结束或低频同步给 Vue
- 下沉本地面向预览
  - 当前标准模式右键拖拽会高频写入 Pinia 的 `facingPreview`
  - 可评估把纯预览面向放到渲染层，服务端输入发送仍保持 50ms 间隔和角度阈值
- 拆分本机连续位姿预览与权威快照
  - 当前连续移动会以 20Hz 修改 Vue store 中的角色位姿
  - 可评估让权威快照保持低频响应式，本机连续位姿预览使用非响应式渲染状态或轻量镜像
- 保持远端角色插值在 Pixi 内部完成
  - 当前 `renderActors` 使用普通 `Map`，方向正确
  - 后续新增动画、倒计时视觉效果和纯渲染状态时，不默认放入 Vue reactive
