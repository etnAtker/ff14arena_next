## 4. 技能 CD 显示

目标是让主动技能在前端具备明确的 CD 展示，并保持服务端权威。

待实现项：

- 在技能按钮上显示 CD 状态：
  - 就绪时显示 `防击退（1）`、`疾跑（2）`
  - 冷却中显示 `防击退 12.3s`、`疾跑 42.8s`
- 按钮禁用条件同时检查：
  - 当前处于 `running`
  - 当前客户端有受控角色
  - 对应技能的 `cooldown.readyAt` 不晚于当前显示模拟时间
- 玩家按下技能时先执行本地乐观 CD：
  - 防击退使用现有 `KNOCKBACK_IMMUNE_COOLDOWN_MS`
  - 疾跑使用现有 `SPRINT_COOLDOWN_MS`
- 每次收到 `sim:snapshot` 时，直接使用服务端快照中的权威 CD 覆盖本地乐观 CD
- 避免在 Vue 组件中散落技能 CD 时长，优先复用现有共享常量或稳定导出

## 5. Vue 高频刷新优化

目标是减少不必要的 Vue 响应式高频写入，把逐帧绘制和纯预览状态尽量留在 Pixi 或非响应式状态中。

建议优化点：

- 降低 HUD 显示时钟刷新频率
  - 当前 `BattlePage` 每帧写入 `renderNowMs`
  - 状态倒计时、技能 CD 和读条文本不需要 60fps 刷新
  - 可评估改为 `100ms` 或 `200ms` 的 HUD tick
- 减少拖拽镜头时的 Vue 更新
  - 当前拖拽镜头时 `cameraYaw` 会按 RAF 写入 Vue ref
  - 可评估把拖拽中的镜头角度放在 `BattleStage` 内部非响应式变量，结束或低频同步给 Vue
- 合并滚轮缩放更新
  - 当前每个 `wheel` 事件都会直接更新 `cameraZoom`
  - 可评估使用 RAF 合并或最小变化阈值
- 下沉本地面向预览
  - 当前标准模式右键拖拽会高频写入 Pinia 的 `facingPreview`
  - 可评估把纯预览面向放到渲染层，服务端输入发送仍保持 50ms 间隔和角度阈值
- 拆分本机连续位姿预览与权威快照
  - 当前连续移动会以 20Hz 修改 Vue store 中的角色位姿
  - 可评估让权威快照保持低频响应式，本机连续位姿预览使用非响应式渲染状态或轻量镜像
- 保持远端角色插值在 Pixi 内部完成
  - 当前 `renderActors` 使用普通 `Map`，方向正确
  - 后续新增动画、倒计时视觉效果和纯渲染状态时，不默认放入 Vue reactive
